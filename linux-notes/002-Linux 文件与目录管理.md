# 第 6 章 · Linux 文件与目录管理

> 核心一句话:**本章是 Linux 日常操作的主菜——在目录树里穿梭、复制、移动、删除文件,以及用 find 精准搜索。** 三大块:① 目录与路径 ② 文件/目录操作命令 ③ 文件查找。

---

## 一、目录与路径

### 1. 特殊目录符号

| 符号 | 含义 |
|------|------|
| `.` | 当前目录 |
| `..` | 上一层目录 |
| `~` | 当前用户家目录 |
| `~account` | 指定账号的家目录 |
| `-` | 上一个工作目录(`cd -` 回到刚才那) |

### 2. 绝对路径 vs 相对路径

| 类型 | 写法 | 特点 |
|------|------|------|
| **绝对路径** | 从 `/` 写起,如 `/home/user/file` | 任何位置都通用,不易出错 |
| **相对路径** | 相对当前目录,如 `../dir/file` | 简短,但依赖当前位置 |

> ⚠️ 易错:`./run.sh` 表示"当前目录的 run.sh",不是"运行 run.sh"。Linux 默认不在当前目录找可执行文件(出于安全),所以执行当前目录的脚本必须加 `./`。

### 3. 路径相关命令

| 命令 | 作用 | 常用选项 |
|------|------|----------|
| `cd` | 切换目录 | `cd -` 回上一个;`cd ~` 回家;`cd` 回家 |
| `pwd` | 显示当前绝对路径 | `-P` 显示真实物理路径(非软链接) |
| `mkdir` | 新建目录 | `-p` 递归建父目录;`-m` 设权限 |
| `rmdir` | 删除**空**目录 | `-p` 连空父目录一起删 |

```bash
mkdir -p a/b/c          # 递归创建多层目录(常用)
mkdir -m 700 secret     # 建目录时直接设权限
rmdir emptydir          # 只能删空目录,非空报错
```

## 二、文件与目录操作命令(重点)

### 1. `cp` —— 复制(最易错,重点掌握)

```bash
cp 来源 目标
```

**关键选项:**

| 选项 | 作用 |
|------|------|
| `-r` / `-R` | 递归复制(复制**目录**必须加) |
| `-i` | 目标已存在时询问确认 |
| `-p` | 保留原文件属性(权限/时间/拥有者) |
| `-a` | = `-r` + `-p`,完整归档复制(最常用) |
| `-u` | 仅当来源比目标新才复制(增量备份) |
| `-f` | 强制覆盖 |

**三大注意点(易错!):**

1. **复制目录必须加 `-r` 或 `-a`**,否则报错。
2. **复制会保留来源文件的拥有者/群组吗?** 默认**不会**——复制后目标文件的拥有者变成执行 cp 的人。要保留用 `-p` 或 `-a`。
3. **`-i` 与 `-f` 优先级**:默认很多系统 alias 了 `cp = cp -i`,所以即使不加 `-i` 也会问你。要强制不问用 `\cp` 或 `command cp` 绕过别名。
4. **复制给他人前**要 `chown` 改拥有者,否则对方可能无权使用。

```bash
cp file.txt /tmp/              # 复制文件到 /tmp
cp -a dir1 dir2                # 完整复制目录(含权限,最常用)
cp -u file.txt /backup/        # 只复制更新过的(增量)
```

### 2. `mv` —— 移动 / 重命名

```bash
mv 来源 目标
```

| 选项 | 作用 |
|------|------|
| `-i` | 覆盖前询问 |
| `-f` | 强制覆盖(默认) |
| `-u` | 仅当来源更新才覆盖 |

**特点:**
- 同一目录下 mv = **重命名**;跨目录 mv = **移动**。
- mv 不需要 `-r` 就能移动目录(和 cp 不同!)。
- mv 跨文件系统(分区)时实际上是"复制+删除",会慢一些。

```bash
mv old.txt new.txt             # 重命名
mv file.txt /tmp/              # 移动
mv dir1 /tmp/                  # 移动目录,无需 -r
```

### 3. `rm` —— 删除(危险!谨慎!)

```bash
rm 文件
```

| 选项 | 作用 |
|------|------|
| `-r` | 递归删除(删**目录**必须加) |
| `-f` | 强制,不询问 |
| `-i` | 删除前逐一询问(安全) |

**注意事项(血泪):**
1. **删目录必须 `-r`**,否则报错。
2. `rm -rf` 是核武器,**一旦路径写错(如 `rm -rf /` 或多打空格 `rm -rf / tmp`)后果不可逆**,Linux 没有回收站。
3. 建议养成习惯:**先 `ls` 看一眼要删的路径,再 rm**。
4. 系统通常 alias 了 `rm = rm -i`,要免确认用 `\rm` 或加 `-f`。

```bash
rm file.txt                    # 删文件
rm -r dir1                     # 删目录及其内容
rm -i *.txt                    # 逐个确认删除(安全)
```

## 三、文件内容查看(基础,后续章节展开)

| 命令 | 作用 |
|------|------|
| `cat` | 正向显示全部内容(适合短文件) |
| `tac` | 反向显示 |
| `nl` | 显示并加行号 |
| `more` / `less` | 分页查看(less 更强,可前翻) |
| `head -n 20` | 看前 20 行 |
| `tail -n 20` | 看后 20 行;`-f` 持续追踪(看日志常用) |

```bash
tail -f /var/log/messages      # 实时盯日志(运维常用)
head -n 10 file.txt            # 前 10 行
less file.txt                  # 分页查看,q 退出,/ 搜索
```

## 四、文件查找 —— find(本章重点)

### 1. 基本语法

```bash
find 路径 [选项] [动作]
```
在指定路径下递归搜索符合条件的文件。

### 2. 常用查找条件

| 选项 | 含义 | 示例 |
|------|------|------|
| `-name` | 按文件名(支持通配符) | `find / -name "*.log"` |
| `-iname` | 文件名忽略大小写 | `find . -iname "readme"` |
| `-type` | 按类型:`f`文件/`d`目录/`l`链接 | `find / -type d -name "test"` |
| `-size` | 按大小:`+`大于/`-`小于/无等于 | `find . -size +10M` |
| `-mtime` | 按修改时间(天):`-n` n天内,`+n` n天前 | `find . -mtime -7` |
| `-mmin` | 按修改时间(分钟) | `find . -mmin -30` |
| `-user` | 按拥有者 | `find / -user alice` |
| `-group` | 按群组 | `find / -group dev` |
| `-perm` | 按权限 | `find . -perm 755` |
| `-uid` / `-gid` | 按 UID/GID | |

### 3. 时间参数详解(易混重点)

`-mtime n` 的含义(以天为单位):
- `-mtime 0`:今天(24 小时内)修改过的
- `-mtime -3`:3 天内修改过(<3 天)
- `-mtime +3`:3 天前修改过(>3 天)
- `-mtime 3`:正好第 3~4 天前之间

> 记忆:`-` 是"以内",`+` 是"以前"。`-mtime -7` = 最近一周。

### 4. 通配符配合

```bash
find /var -name "*.log"                 # 所有 .log 文件
find . -name "[A-Z]*"                   # 大写字母开头的文件
find / -name "passwd"                   # 全盘找 passwd
```

### 5. 找到后的动作

| 动作 | 作用 |
|------|------|
| `-print` | 默认,打印路径 |
| `-ls` | 以 ls -l 格式输出 |
| `-exec 命令 {} \;` | 对找到的每个文件执行命令 |
| `-delete` | 删除找到的文件 |

```bash
# 找出所有 .log 文件并删除
find /tmp -name "*.log" -exec rm {} \;

# 找出大于 100M 的文件,看详情
find / -type f -size +100M -exec ls -lh {} \;

# 找出 7 天前的 .tmp 文件并删除
find /tmp -name "*.tmp" -mtime +7 -delete
```

> `{}` 代表找到的文件,`\;` 是 -exec 的结束符。`\;` 每个文件执行一次命令,`+` 可批量传参(更快)。

### 6. find vs 其他查找命令

| 命令 | 查找对象 | 速度 | 特点 |
|------|----------|------|------|
| `find` | 全盘扫描文件系统 | 慢 | 条件最丰富,最强大 |
| `locate` | 查数据库索引 | 极快 | 需 `updatedb` 更新,可能滞后 |
| `which` | PATH 中的可执行命令 | 快 | 只找命令位置 |
| `whereis` | 命令+源码+man | 快 | 限定在系统目录 |

```bash
locate passwd              # 秒级出结果(查数据库)
which python3              # 找命令在哪
```

## 五、其他实用命令

| 命令 | 作用 |
|------|------|
| `basename 路径` | 取文件名(`/a/b/c.txt` → `c.txt`) |
| `dirname 路径` | 取目录名(`/a/b/c.txt` → `/a/b`) |
| `touch file` | 新建空文件 / 更新时间戳 |
| `file file` | 判断文件类型(不靠扩展名) |
| `stat file` | 看文件详细时间戳和属性 |

```bash
basename /home/user/file.txt    # 输出 file.txt
dirname /home/user/file.txt     # 输出 /home/user
file unknownfile                # 告诉你这是文本/二进制/压缩包...
```

## 六、重点速记清单

1. **特殊目录**:`.`/`..`/`~`/`-`,执行当前目录脚本要 `./`。
2. **cp 三大坑**:复制目录加 `-r`/`-a`;默认不改属性需 `-p`;复制给他人要 `chown`。
3. **mv**:同目录=重命名,跨目录=移动,移动目录**无需** `-r`。
4. **rm**:删目录加 `-r`,`rm -rf` 是核武器,先 ls 再删。
5. **find 语法**:`find 路径 条件 动作`,条件可组合。
6. **find 时间**:`-mtime -n`=n天内,`+n`=n天前;`-`内 `+`前。
7. **find 动作**:`-exec ... {} \;` 对结果逐个执行命令。
8. **四大查找**:find(全盘,慢但强)、locate(查库,快但可能旧)、which(找命令)、whereis(找命令+文档)。
9. **tail -f** 看实时日志,运维必备。

## 七、常见易错点回顾

- ❌ 以为执行当前目录脚本直接打文件名 → 必须加 `./`,如 `./run.sh`。
- ❌ 复制目录忘了 `-r` → 报 `cp: omitting directory`。
- ❌ 以为 cp 会保留拥有者 → 默认不保留,目标拥有者变成你自己。
- ❌ `rmdir` 删非空目录 → 只能删空目录,非空用 `rm -r`。
- ❌ `rm -rf` 路径写错或多打空格 → 灾难,先 `ls` 确认再删。
- ❌ find 的 `-mtime +7` 理解成"7 天内" → 实际是"7 天前",`-` 才是"内"。
- ❌ find 全盘搜索太慢 → 可先用 `locate` 或限定路径范围提速。
